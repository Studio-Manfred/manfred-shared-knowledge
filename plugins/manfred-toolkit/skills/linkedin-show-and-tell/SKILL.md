# LinkedIn — Show & Tell Post (Jens Wedin / Manfred)

## Overview

En show & tell post presenterar något verkligt — en demo, ett resultat, ett verktyg i action. Mindre om att lära ut hela processen och mer om att väcka nyfikenhet: *titta vad som är möjligt*. Mediet (video, bild) gör det tunga lyftet — texten skapar kontext och känsla.

## When to Use

- User wants a LinkedIn post showcasing something concrete — a demo, tool, result, or output
- Post has or should have attached media (video, screenshot, GIF)
- The visual/media carries the weight; text provides context and reaction

**When NOT to use:**
- Post is primarily teaching a concept → use `manfred-toolkit:linkedin-teach`
- Post is a reflection or celebration without concrete artifact → use `manfred-toolkit:linkedin-reflect`
- Post should be in English or for a different person/brand

## Voice & Tone

- **Språk:** Alltid svenska
- **Stil:** Exalterad men jordnära, nördig, varm, direkt, konversationell
- Låt genuin spänning synas — *jäklar*, *nyp mig i armen*, *tänk att få...*
- **Namnge alltid verktygen, personerna och företagen** — det ger trovärdighet och värme
- Korta meningar. Snabbt tempo. Låt demon tala.
- Testa alltid: *Skulle jag klicka på detta? Är det häftigt? Berättar det en story?*

## Post Structure

1. **Hook** — Vad tittar vi på? En snabb rubrik eller punchig öppningsrad.
2. **Kontext** — Varför finns detta? Vilket problem löser det, eller vad fick dig att göra det?
3. **Vad du visar** — Beskriv vad som finns i videon/bilden på ett sätt som tillför kontext, inte bara bildtext. Nördiga detaljer välkomna.
4. **Reaktion/Känsla** — Din ärliga reaktion. Jäklar-ögonblick. *Tänk att få...*
5. **Fråga eller inbjudan (valfri)** — Bjud in reaktioner eller samtal.
6. **Signatur** — Avsluta alltid med:

```
〰️
Hej, jag heter Jens och driver Manfred. [en mening anpassad till vad som visades].
```

## Length & Format

- 100–200 ord — kortare än teach/reflect, mediet bär tyngden
- Snabba stycken, inga textväggar
- Max 1–2 emoji för struktur
- Undvik att förklara för mycket — lita på det visuella

## Example

> Från skiss till design system till ny app — på en lunch.
>
> Axel Nathorst-Böös tog fram en ny brand-skiss för Manfred. Jag bad Claude bygga ett design system i Storybook baserat på den.
>
> I filmen nedan kan du se vad som händer när jag sen pekar Claude mot kodbiblioteket i Pencil.dev och ber den rita upp en ny app. Allt baserat på vårt design system. Inga genvägar.
>
> Jäklar vad det känns bra när det funkar. Tänk att det faktiskt går att jobba så här idag.
>
> Hur använder du AI i ditt designarbete?
>
> 〰️
> Hej, jag heter Jens och driver Manfred. Vi hjälper företag bygga design och researchförmåga på ett modernt sätt.

## Common Mistakes

- Beskriver demon istället för att uppleva den → skriv från reaktion, inte beskrivning
- Begravd hook → första raden måste få folk att stanna i scrollandet
- Glömmer att namnge verktygen/personerna → *Pencil.dev*, *Claude*, *Axel* ger trovärdighet och värme
- Generisk spänning ("Det här är amazing!") → var specifik om vad som är amazing
- Gör den för lång → mediet är stjärnan, texten är ramen
- Signatur inte anpassad → vad gör Manfred som relaterar till det som just visades?
